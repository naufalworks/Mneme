Q1: "How does it work?"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

It's a storage system (like a database) that:
  1. You tell it what to store (project info, why it exists)
  2. It saves to files on disk (.claude/knowledge/)
  3. Later, you ask it questions
  4. It reads the files and gives you answers

NO LLM API needed. Just Python reading/writing files.


Q2: "What file do I run to start using it?"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

To see it work:
  $ python3 working_demo.py

To use in your code:
  from claude_code_integration import init_knowledge_system
  knowledge = init_knowledge_system()


Q3: "How do I know it's working and not using Claude session?"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PROOF: It creates REAL FILES on disk!

Run this:
  $ python3 working_demo.py
  $ ls -la .claude/knowledge/

You'll see:
  .claude/knowledge/
  ├── system_state.json          ← System state
  ├── knowledge_base.json        ← Hypervector data
  └── projects/
      └── rate-limiter/
          ├── .meta/origin.json  ← Machine-readable
          └── docs/WHY.md        ← Human-readable

Read the WHY.md file:
  $ cat .claude/knowledge/projects/rate-limiter/docs/WHY.md

You'll see the "why" information stored as TEXT in a FILE.
NOT in Claude's session memory!

  • global_knowledge_system.py
  • hypervector.py
  • neuromorphic_agent.py
  • crystallization.py

Step 2: In your Python code:
  from claude_code_integration import init_knowledge_system
  knowledge = init_knowledge_system()

Step 3: Track projects as you create them:

      reason="What it does",
      created_for="parent-service"  # If created for another project
  )

  print(knowledge.why_does_exist("my-service"))
